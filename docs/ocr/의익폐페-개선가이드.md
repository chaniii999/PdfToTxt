# 의/익/폐/페 오인식 개선 가이드

획 뭉개짐, 분절 실패, 언어모델 보정 약함 대응.

## 적용 순서 (ROI 최소 변경 기준)

1. **300DPI 유지 + 업스케일(1.5~2x)** ✅
2. **2-pass: 전체 PSM 6 → 저신뢰/의심 토큰 ROI만 PSM 10/8 재시도** ✅
3. **user_words + user_patterns** ✅
4. (필드 한정) whitelist
5. (필요 시) 폰트 기반 추가 학습 traineddata

## 1. 입력 품질: 획 보존

| 항목 | 환경변수 | 기본값 | 설명 |
|------|----------|--------|------|
| 업스케일 | `OCR_UPSCALE` | 1.5 | 1.5~2.0x. LSTM이 획을 더 잘 잡게 함 |
| Gray bypass | `OCR_USE_GRAY` | 0 | 1이면 Otsu 대신 gray 그대로. 과한 이진화 시 ㅢ/ㅔ 보존 |

## 2. 2-pass PSM

| 환경변수 | 값 | 설명 |
|----------|-----|------|
| `OCR_2PASS` | 1 | conf < 70 또는 의/익/폐/페 포함 짧은 토큰 시 ROI 재시도 |

- 1차: PSM 6 (문단)
- 2차: ROI crop 후 PSM 10(한 글자) 또는 PSM 8(단어)

## 3. user_words, user_patterns

- `config/tesseract/user_words.txt`: 원본, 의견, 익명, 폐기, 폐지 등
- `config/tesseract/user_patterns.txt`: 원본:, 추출:, 의:, 폐: 등

문서 도메인에 맞게 실제 등장 단어를 추가할 것.

## 4. 구현 파일

- `services/ocr/preprocess_minimal.py`: 업스케일, gray bypass
- `services/ocr/pdf_ocr.py`: TESS_CONFIG, user_words/patterns, OCR_2PASS
- `services/ocr/ocr_twopass.py`: 2-pass ROI 재시도
