# 한글 자모 혼동 규칙 (조합 구조 기반)

사전 없이 한글 조합 방식(초성·중성·종성)을 분석하여 인식 보정.

## 1. [원본:추출] 형식

- **원본**: 정답(ground truth)
- **추출**: OCR 오인식 결과
- 보정 방향: 추출 → 원본

## 2. 피드백 — 자모 구조 분석

| 원본 | 추출 | 적용 | 비고 |
|------|------|------|------|
| 성 | 섬 | ✅ | 종성 ㅇ↔ㅁ |
| 많 | 않 | ✅ | 초성 ㅇ↔ㅁ |
| 의 | 익 | ❌ | LLM에서 교정 |
| 폐 | 페 | ❌ | LLM에서 교정 |

### 2.1 성:섬

- **성**: ㅅ + ㅓ + ㅇ
- **섬**: ㅅ + ㅓ + ㅁ
- ㅇ(원형)과 ㅁ(사각형)은 이진화 시 경계가 흐리면 혼동

### 2.2 익:의

- **의**: ㅇ + ㅢ(ㅡ+ㅣ) + 빈
- **익**: ㅇ + ㅣ + ㄱ
- ㅡ(가로선)가 약하면 ㄱ(꺾임)처럼 보임

### 2.3 많:않

- **많**: ㅁ + ㅏ + ㄶ
- **않**: ㅇ + ㅏ + ㄶ
- 초성 ㅇ↔ㅁ (성:섬과 동일 자모, 위치만 다름)

### 2.4 페:폐

- **폐**: ㅍ + ㅖ + 빈
- **페**: ㅍ + ㅔ + 빈
- ㅖ = ㅕ+ㅣ (상단 가로획). ㅔ = ㅓ+ㅣ. 얇은 가로획 손실 시 ㅖ→ㅔ

## 3. 적용 규칙 (구조 기반)

| 조건 (추출 구조) | 보정 |
|-----------------|------|
| ㅅ+ㅓ+ㅁ | → ㅅ+ㅓ+ㅇ (성) |
| ㅇ+ㅏ+ㄶ | → ㅁ+ㅏ+ㄶ (많) |
| ㅇ+ㅣ+ㄱ, ㅍ+ㅔ | LLM 단계에서 교정 |

## 4. 전처리 연계

- **CLAHE** (clip_limit 2.5): 얇은 획(ㅖ 가로선) 보존
- **Sharpen** (0.55): ㅇ/ㅁ, ㅔ/ㅖ 경계 선명화
- **Morphology** (2×2 close): 끊긴 획 연결

## 5. 잘못 나뉜 음절 병합

OCR이 줄바꿈/공백으로 한글 음절을 잘못 나눈 경우 복원.

| 추출 | 원본 | 조건 |
|------|------|------|
| 경\n우 | 경우 | 한글 + 줄바꿈/2칸+공백 + 한글, 두 음절이 다를 때 |
| 경  우 | 경우 | 동일 |
| 한다\n다음 | (병합 안 함) | 다=다 이므로 제외 |

## 6. 구현

- `services/ocr/jamo.py`: 자모 분리·조합
- `services/ocr/postprocess.py`: 규칙 기반 보정 + 잘못 나뉜 음절 병합
