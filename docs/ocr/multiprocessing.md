# OCR 멀티프로세싱

Tesseract는 이미지 1장씩만 처리하므로, ProcessPoolExecutor로 페이지별 병렬화.

---

## 설정 (i5-6600 4코어 기준)

| 환경변수 | 기본값 | 설명 |
|----------|--------|------|
| `OCR_USE_MULTIPROCESS` | 1 | 0이면 순차 처리 |
| `OCR_MAX_WORKERS` | 3 | 프로세스 수. i5-6600(4코어)에선 3 권장 |

- **3 workers**: 메인 프로세스·시스템 여유 1코어
- 4 workers도 가능하나, I/O·메모리 여유를 위해 3 권장

---

## 동작

- `total >= 2`이고 `OCR_USE_MULTIPROCESS=1`일 때만 병렬 처리
- 1페이지 PDF는 순차 처리 (오버헤드 방지)
- `ex.map(..., chunksize=1)`: 페이지 순서 유지

---

## 효과

- 4코어 CPU: 이론상 ~3배 속도 향상
- 50페이지 PDF: 순차 ~500초 → 병렬 ~170초 (가정)
